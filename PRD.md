# AI 日报 Agent - 产品需求文档

## 1. 产品概述

一个部署在 Vercel 上的 AI 日报网页应用，用户手动触发后自动抓取前一天各大 AI 博客的 RSS 内容，通过 AI 进行分类、去重、生成中文摘要，以日报形式展示在页面上，并支持导出为 Markdown 文件。

## 2. 技术栈

- 框架：Next.js
- 开发语言：TypeScript
- 部署：Vercel
- AI 服务：通过环境变量适配 OpenAI / Anthropic / Gemini 三种请求格式

## 3. 数据源

从以下高质量 AI 博客的 RSS 获取内容：

**国外：**
- OpenAI Blog（https://openai.com/blog/rss.xml）
- Google AI Blog（https://blog.google/technology/ai/rss/）
- Hugging Face Blog（https://huggingface.co/blog/feed.xml）

**国内：**
- 机器之心（https://www.jiqizhixin.com/rss）
- 量子位（https://www.qbitai.com/feed）
- 36kr（https://36kr.com/feed）

说明：Anthropic Blog 和 Meta AI Blog 当前未提供可稳定访问的官方 RSS 地址，不纳入 RSS 自动抓取源。

## 4. 分类规则

### 4.1 领域分类

| 分类 | 说明 |
|------|------|
| 大模型 | 大语言模型相关的进展、发布、评测 |
| 图像生成 | 图像/视频生成模型和工具 |
| AI 编程 | 代码生成、编程辅助工具 |
| AI 软件技术及知识 | AI 相关的工程实践、架构、知识 |

### 4.2 类型标签

- 产品发布
- 教程/实践

### 4.3 重要程度

- 重大事件
- 一般动态

### 4.4 数量限制

- 每个分类最多展示 15 条内容
- 总分类数不超过 5 类
- AI 需对相似内容进行去重

## 5. 内容输出格式

每条内容包含：

- **标题**：中文
- **摘要**：一段中文总结（3-5 句话概括文章核心内容）
- **原文链接**：可点击跳转至原文

所有内容统一以中文展示，英文原文由 AI 翻译后输出。

- 浅色主题
- 极简日报风格
- 列表布局，按分类分组展示
- 信息密度优先，注重可读性

### 6.3 交互说明

- 无历史记录功能，每次打开页面为空白状态
- 当天没有内容的分类不显示

## 7. 功能说明

### 7.1 手动抓取

- 页面提供"抓取今日内容"按钮
- 点击后展示进度条，显示当前状态（正在抓取、正在分类、完成）
- 某个 RSS 源抓取失败时跳过该源继续处理其他源
- 失败后在页面上提示具体失败原因

### 7.2 导出 Markdown

- 页面提供"导出 MD"按钮
- 点击后直接下载当前页面全部内容为 `.md` 文件
- 文件命名格式：`2026-06-15-ai-daily.md`

## 8. 环境变量配置

用户在 Vercel 环境变量中配置 AI 服务，支持以下三种格式任选其一：

| 变量名 | 说明 |
|--------|------|
| `AI_PROVIDER` | 选择服务商：`openai` / `anthropic` / `gemini` |
| `AI_API_KEY` | 对应服务商的 API Key |
| `AI_MODEL` | 使用的模型名称（如 `gpt-4o`、`claude-sonnet-4-20250514`、`gemini-pro`） |
| `AI_BASE_URL` | 可选，自定义 API 地址（用于代理或兼容接口） |

## 9. 核心流程

```
用户点击"抓取今日内容"
    ↓
并发请求所有 RSS 源（失败的跳过）
    ↓
解析 RSS，提取前一天发布的文章标题和内容
    ↓
将文章内容发送给 AI 服务
    ↓
AI 执行：去重 → 分类 → 生成中文摘要
    ↓
返回结构化数据，渲染到页面
    ↓
用户可选择导出为 MD 文件
```

## 10. 非功能需求

- 单次抓取流程应在 60 秒内完成
- 页面响应式布局，支持移动端浏览
- 无需用户登录，无需数据库，无需持久化存储
